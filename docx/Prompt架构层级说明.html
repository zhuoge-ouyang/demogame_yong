🧠 AI Prompt 架构层级说明

西幻塔防世界观项目 · Prompt 调用关系全景图

一、系统提示词三层结构

每次 AI 生成时，根据模块类别注入不同的系统提示词层级组合：

LAYER 1: BASE（始终注入）
  • 纯西幻风格约束（禁止东方修仙/武侠）
  • 逻辑自洽要求
  • 塔防玩法融合要求
  • 文风：简洁有力、画面感、中文输出
    • LAYER 2: SETTING（设定模块注入）
  • 三界基础结构（上界/凡界/深渊）
  • 世界树：尤克特拉希尔
  • 九封魂令 · 九位英雄
  • 核心世界观框架
    • LAYER 3: NARRATIVE（剧情模块注入）
  • 玩家身份：种子灵魂 / 方舟堡 / 根语者
  • 方舟堡根脉迁移机制
  • 神殿五大派系
  • 关键角色弧线
  • 禁止项：凡界不涉及第十魂令

    • 模块类别 → 层级注入映射

    setting 设定模块

    realm-overview · realm-upper · realm-mortal · realm-abyss
    → 注入：BASE + SETTING

    worldview 世界观模块

    world-tree-system · phase2-主题表达 · phase2-游戏内表达
    → 注入：BASE + SETTING

    storyline 剧情模块

    main-storyline · phase2-主线/冲突/目标/体验 · phase3-entry/completion/levels
    → 注入：BASE + NARRATIVE

    character 人设模块

    player-identity · hero-system · castle-goddess · phase3-boss
    → 注入：BASE + NARRATIVE

    二、用户点击"AI 生成"后的完整流程

    1 获取 Prompt 模板
    检查是否有用户自定义覆盖 → 否则使用原始模板（phase1/2/3-prompts.ts）
    2 确定模块类别
    setting / worldview / storyline / character → 决定注入哪些系统提示词层
    3 选择系统提示词
    根据类别拼接 LAYER_BASE + LAYER_SETTING 或 LAYER_NARRATIVE
    4 收集上下文依赖
    从 world.json 中提取 contextDependencies 指定的已定稿/草稿数据 + 注入 reference 参考文档
    5 智能裁剪
    按优先级裁剪：定稿(不裁) > 草稿(段落裁) > 参考(优先裁) · 保持 Markdown 语义完整
    6 根据模式组装用户 Prompt
    全新生成 / 续写 / 扩写 — 不同模式拼接不同的指令和内容块
    7 发送给 AI 并返回结果
    system + user → AI 服务 → 返回生成内容 → 填充到编辑区

    三、三种生成模式对比

    🆕 全新生成 (Full)

    场景：初次创建，或推翻重写
    • 已定稿内容 → 标记"严格遵循"
    • 草稿内容 → 标记"可优化"
    • 参考文档 → 标记"可借鉴"
    • 输出：完整模块内容

    📝 续写模式 (Continue)

    场景：部分字段为空，需要补完
    • 已定稿内容 → "不可修改"
    • 空白字段 → "请补充完整"
    • 草稿 → 作为参考
    • 输出：仅待续写部分

    🔄 扩写模式 (Expand)

    场景：内容已有，需要深化
    • 所有现有内容 → 完整展示
    • 扩展要求 → 用户指定方向
    • 保留全部原有内容
    • 输出：原内容 + 新增深化

    七、完整案例：生成"深渊"模块

    📋 案例：用户在 realm-abyss 页面点击"AI 生成"（全新生成模式）
    Step 1 模板：使用 phase1-prompts.ts 中的 'realm-abyss' 模板 Step 2 类别：setting → 注入 BASE + SETTING 层 Step 3 系统提示词： ┌─ LAYER_BASE：西幻风格约束 + 文风要求 └─ LAYER_SETTING：三界结构 + 世界树 + 九封魂令 Step 4 上下文依赖：contextDependencies = [realm-overview, realm-upper, realm-mortal] ┌─ [已确认] 三界概述（定稿内容，完整注入） ├─ [已确认] 上界·神殿（定稿内容，完整注入） ├─ [参考] 凡界（草稿内容，标记可优化） └─ [参考素材] abyss-realm-refs.ts 的3篇文档（过去/现状/未来） Step 5 裁剪：如果总 Token 超出60%预算 → 优先裁剪参考素材 → 其次草稿 Step 6 组装用户 Prompt： ┌─ userPromptTemplate（深渊设计指令） ├─ === 已确认的设定 ===（三界概述+上界 完整内容） ├─ === 参考内容 ===（凡界 草稿内容） ├─ === 参考素材 ===（3篇深渊参考文档） ├─ 【重要约束】对已确认设定严格遵循 └─ 【输出格式要求】按指定格式输出 发送给 AI：system（~800 tokens）+ user（~4000 tokens）= ~5000 tokens
    📋 案例：续写模式 — 上界已定稿过去+现状，需要续写"未来"
    模式：continue（续写） 系统提示词：同上（BASE + SETTING） 用户 Prompt 结构： ┌─ "你是世界观续写助手，请续写「上界·神殿」模块" ├─ === 已确认内容（不可修改）=== │ ├─ 上界-过去：完整定稿内容 │ └─ 上界-现状：完整定稿内容 ├─ === 待续写部分 === │ └─ 上界-未来：（当前为空，请补充完整内容） ├─ === 参考素材 === upper-realm-refs 未来部分 └─ "请仅输出待续写部分，确保与已确认内容逻辑连贯" 关键区别：AI 只输出"未来"部分，不重复已有的"过去"和"现状"

    八、上下文优先级与裁剪机制

    优先级内容类型裁剪策略说明
    critical（最高）已定稿的上游模块完全保留，不裁剪AI 必须严格遵循
    high草稿内容段落裁剪至~400字AI 可在此基础上优化
    medium参考文档段落裁剪至~200字AI 可借鉴风格方向
    low远距离依赖完全移除仅保留"已省略"标记
